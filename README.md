# SF6 Move Galleries

Frame-accurate animated galleries of Street Fighter 6 movesets, captured in
Training mode with the REFramework hitbox overlay (green hurtboxes, pink
throw/active boxes).

**[▶ View the galleries](https://lowekeyflexin.github.io/sf6-moves/)**
*(once GitHub Pages is enabled — see below)*

## Structure

```
/
├── index.html        ← character picker (the landing page)
├── ingrid/
│   ├── index.html    ← Ingrid's gallery
│   └── gifs/         ← her move clips
└── <future char>/    ← same layout per character
```

The root `index.html` lists every character folder present, so adding a new
character is just: build their `<char>/` folder and they appear on the landing
page automatically. Ingrid is the only character today.

Each gallery is self-contained (no server, no build step): native
`<details>` collapse, all categories closed by default, and each clip uses
`loading="lazy"` so a gif downloads only when its section is opened. A
single-active-tab guard (BroadcastChannel) unloads clips in older tabs so only
one tab ever holds the heavy gifs in memory.

## Coverage (Ingrid)

Normals (standing/crouching/jumping), command normals + target combos, throws,
Drive system, specials (Sun Shot, Sun Flare, Solar Burst, Sun Veil, Vanishing
Sun) including **Sun Crest variants** (no-crest / 1-crest / 2-crest), and supers
(Shining Sun SA1, Order of the Sun SA2, Cosmic Ray SA3 on block / on hit / CA
cinematic) with their crest-consume variants.

## Regenerating

Generated by `build_static_gallery.py` from the gallery manifest, so the export
always matches the current move set / crops:

```bash
python z_source/scripts/sf6_coach/build_static_gallery.py
```

## GitHub Pages setup

1. Push this folder to a new repo (e.g. `sf6-moves`).
2. Repo → **Settings → Pages** → Source `Deploy from a branch`, branch `main`,
   folder `/ (root)`.
3. ~1 min later the picker is live at
   `https://lowekeyflexin.github.io/sf6-moves/`.
